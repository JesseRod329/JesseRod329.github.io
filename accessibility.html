Accessibility

This site is designed to be usable with keyboard, screen readers, and high-contrast modes. If you encounter any barriers, please email jesse@jesserodriguez.me.

Commitment

I aim to meet or exceed WCAG 2.2 AA where practical for a static site, while preserving performance, privacy, and security. The terminal interface is progressively enhanced and fully functional without JavaScript via the on-page navigation and classic versions.

Key Accessibility Features

  • Semantic HTML structure with landmarks: header, main, footer.
  • Logical heading hierarchy and descriptive link text.
  • Keyboard support with visible focus states and skip link.
  • Reduced motion by respecting user preferences; minimal entrance animations.
  • Color contrast targeting WCAG AA for text and interactive elements.
  • No reliance on color alone to convey meaning.
  • Responsive layout supporting zoom to 200% without loss of content.
  • Progressive enhancement: noscript fallbacks and classic/ios versions.
  • Privacy-first: all tools run client-side; no tracking scripts.

Keyboard Navigation

  • Global: use Tab / Shift+Tab to move focus.
  • CLI: supports Enter, history with ↑/↓, and Tab for autocomplete.
  • Skip link provided to bypass repeated navigation.

Compatibility and Testing

  • Screen readers: VoiceOver (Safari), NVDA (Firefox), Orca (Linux).
  • Browsers: Chrome, Safari, Firefox, and mobile equivalents.
  • Devices: Desktop, iOS (`/ios/`), and a classic desktop version (`/classic.html`).

Feedback

If you find accessibility issues, please email jesse@jesserodriguez.me with details. I aim to respond within 5 business days.
